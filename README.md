# Emotion Training

Тренажёр распознавания эмоций по выражению лица на основе физиогномической традиции. Часть экосистемы инструментов самопознания (рядом с [face](https://face-topaz.vercel.app)).

**Не угадайка.** После каждого ответа — развёрнутый разбор: какие мимические маркеры стоило заметить, чем эта эмоция отличается от похожих, что состояние говорит о человеке с точки зрения физиогномики.

## Стек

- **Next.js 16** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS 4** (single-file PostCSS pipeline)
- **Framer Motion** для микро-анимаций
- **localStorage** для прогрессии (без аккаунтов, без бэкенда)
- **`@google/genai`** только для генерации картинок на этапе сборки — рантайм без AI-вызовов

## Структура

```
src/
  app/
    page.tsx           ← лендинг
    train/page.tsx     ← главный тренировочный цикл
    progress/page.tsx  ← статистика по эмоциям
    layout.tsx, globals.css
  data/
    emotions.ts        ← таксономия 19 эмоций × 3 уровня + FACS-маркеры
    training_set.ts    ← карточки (image_path → emotion_id)
  lib/
    storage.ts         ← localStorage прогресс
    scoring.ts         ← correct/partial/wrong + генерация дистракторов
scripts/
  image-prompts.ts     ← prompts для каждой карточки (анатомически точные, с AU)
  generate-images.ts   ← Nano Banana CLI → public/training/*.jpg
  extract-knowledge.ts ← парсинг отчёта NotebookLM → emotions_analysis.json
public/
  training/            ← сгенерированные лица (не в репо до curation)
```

## Архитектурные решения

### Почему AI-генерация лиц, а не реальный датасет

1. **Лицензии.** KDEF, FACES, AffectNet и другие валидированные эмоциональные датасеты требуют институциональной лицензии — нельзя выложить в публичный веб.
2. **Контроль.** Для различительных пар (дюшенновская vs социальная улыбка, страх vs удивление) нужен точный контроль над FACS Action Units. С AI можно сгенерировать "то же лицо, но с AU6 vs без AU6" для прямого сравнения.
3. **Качество стока.** Бесплатные стоковые эмоции — это поза, не подлинность. Тренажёр на них теряет смысл.

Выбран **Google `gemini-2.5-flash-image` (Nano Banana)**: лучший контроль над тонкими мимическими инструкциями (AU14, асимметрия), $0.04/картинка, character consistency для парных кадров.

### Почему dark theme + serif для разбора

- Dark theme — преемственность с [face-topaz](https://face-topaz.vercel.app).
- Serif (Lora) для блока разбора — заставляет _читать_, а не пробегать глазами. Это серьёзный аналитический продукт, не геймифицированный тренажёр.
- Акцент `#d4a574` (тёплый янтарь) вместо фиолетово-неонового — спокойный, аналитический, для аудитории рекрутеров/HR.

### Почему 3 уровня, а не плоский набор

- **Уровень 1** (базовые 7) — фундамент по Экману. Без этого нет смысла в нюансах.
- **Уровень 2** (различительные пары) — основная работа: страх vs удивление, контролируемый гнев vs презрение, грусть vs усталость. Здесь формируется навык.
- **Уровень 3** (смешанные/подавленные) — высший класс: ностальгия, подавленный гнев под маской, тревога. Открывается только когда стабильно справляетесь с уровнем 2.

Прогрессия: ≥6 ответов на текущем уровне с ≥70% точности → следующий открыт.

### Источник знаний для разбора

Глубина разбора берётся из **NotebookLM-генерации по 18 PDF** в Face notebook (Экман, Хигир, Велховер, Хэннер, Богданов, Тимоти Map, Дюрвиль, Тиклл, Павлов, Розаль, Швар и др.). Сначала генерируется один большой структурированный отчёт через `notebooklm generate report`, затем парсится в `emotions_analysis.json` и подмешивается в `EMOTIONS` map.

## Установка и запуск

```bash
npm install
npm run dev   # http://localhost:3000
```

## Подготовка контента

### 1. Извлечь знания из NotebookLM

```bash
export PATH="$HOME/bin:$PATH"
export PYTHONUTF8=1
notebooklm use 960d2264-94e2-4f92-aaa2-9fac4504635c   # Face notebook
notebooklm artifact list                                # дождаться отчёта
notebooklm download report ./data/raw_knowledge/report.md
npm run extract:knowledge                               # парсит в JSON
```

### 2. Сгенерировать тренировочные картинки

```bash
export GOOGLE_GENAI_API_KEY=...
npm run generate:images                # ~$1.30 на полный сет
npm run generate:images -- --tier 1    # только первый уровень
npm run generate:images -- --only joy-1,joy-2 --force  # перегенерация конкретных
```

После генерации **визуально отсмотрите каждую карточку** — удалите/перегенерируйте те, где эмоция не убедительна. Это ключ к качеству тренажёра.

## Деплой

Vercel: `vercel --prod`. Картинки в `public/training/` и парсенные знания в `src/data/emotions_analysis.json` идут в репо (после curation).

## Roadmap (после MVP)

- AI-judge свободного ответа: пользователь пишет своё описание эмоции → GPT-4o оценивает с парциальным кредитом.
- Видео-карточки (короткие петли с микро-выражениями).
- Mobile через Capacitor (как у [face](https://github.com/sagyndykovmax-svg/face)).
- A/B карточки в одном кадре (две версии одного лица — нужно выбрать настоящую).
